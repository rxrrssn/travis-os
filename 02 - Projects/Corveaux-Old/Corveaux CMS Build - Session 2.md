# Corveaux CMS Build - Session 2

## What Happened

Continuation of Session 1. Session 1 ended with the dev server crashing due to a heap OOM error during Turbopack's initial compilation of `/admin`. This session resolved that crash and verified all routes.

---

## Root Cause: OOM Crash

Turbopack was walking two separate `node_modules` trees simultaneously:

- `C:\Users\razor\documents\github\corveaux\node_modules\` (outer workspace — stub dependencies pre-installed for future auth work)
- `C:\Users\razor\documents\github\corveaux\corveaux\node_modules\` (actual project)

Both directories had `package-lock.json`, which triggered Turbopack's workspace root heuristic and caused it to treat the outer directory as a workspace root. This led to redundant module resolution across both trees and blew out the Node.js heap.

**Error:**
```
FATAL ERROR: NewSpace::EnsureCurrentCapacity Allocation failed - JavaScript heap out of memory
thread 'tokio-runtime-worker' panicked: OS can't spawn worker thread: Insufficient system resources
```

---

## Fix

Two-part fix:

1. **Removed outer `node_modules` and `package-lock.json`** — eliminated the dual-tree resolution issue. The outer `package.json` (next-auth, @auth/prisma-adapter, zod) was kept as a dependency reference for future auth work; only the installed modules were removed.

2. **Increased Node.js heap** — started dev server with `NODE_OPTIONS=--max-old-space-size=4096` to give Turbopack 4GB instead of the default ~1.5GB. Run dev with:

```powershell
$env:NODE_OPTIONS='--max-old-space-size=4096'; npm run dev
```

Or add `NODE_OPTIONS=--max-old-space-size=4096` to `.env` to make it permanent.
