# TDX (TeamDynamix)

## What It Is

Operational workflow and service management platform used as a lightweight CRM and orchestration layer.

## Current Use Cases

- Contact Center interaction records
- Coaching workflows
- Student interaction tracking
- Automated resource delivery
- Workflow automation
- Request routing
- Survey triggers
- CRM-like functionality
- iPaaS flows and Power Automate integrations

## Extended Orchestration Work

TDX is being operated far beyond a traditional ticketing system. Current builds include:
- Workflow-driven student engagement systems
- Automated outreach and resource delivery
- Parent/child lifecycle structures for coaching cadences
- Dynamic resource delivery based on student context
- CRM-like operational logic for student tracking
- Coaching cadence automation

## Known Implementation Patterns

- Parent/child ticket structures
- Cadence-based workflow generation
- Automated student resource emails
- CTI integration experiments (RingCentral)
- Requestor resolution workflows
- Workflow-triggered surveys

## Strategic Goal

Use TDX as:
- Lightweight operational CRM
- Unified orchestration platform
- Centralized operational workflow layer

Reduce: app sprawl, manual routing, process duplication.

## Heavy Usage Areas

- Workflows and forms
- Custom attributes
- API integrations
- Power Automate connections
